## **补充说明**

### 1. 这些文档不是“额外工作”，而是“工作本身”
- **周会纪要**：就是你们开周会时顺便记的，不是额外写。
- **数据周报**：核心数据本来就要看，只是需要固定格式记录下来。
- **需求池整理**：产品经理本来就要梳理需求，只是每周固定同步一次。

### 2. 建议的团队节奏

| 时间          | 例行事务                       | 产出文档                                                    |
| ------------- | ------------------------------ | ----------------------------------------------------------- |
| **周一上午**  | 开周会，同步上周情况、本周计划 | 周会纪要、数据周报、需求池整理                              |
| **周一/周二** | 投放、技术状态同步             | 渠道投放周报、服务器状态周报                                |
| **周四**      | 确定下周话题                   | 话题运营日历（下周）                                        |
| **周五下午**  | 本周复盘，同步进展             | 用户反馈周报、活动效果周报、竞品周报、故障周报、OKR进度周报 |

### 3. 模板化建议
每个周报都建议建立固定模板，每周只需填写数据和变化部分，10-20分钟即可完成。例如：

**核心数据周报模板（示例）**
```
## 一、本周核心数据（8.10-8.16）
- 日活：XX万（环比 +5%）
- 新增：XX万（环比 -2%）
- 次日留存：XX%（环比 +1%）
- 举报率：XX%（环比 -0.5%）

## 二、波动原因分析
- 日活上升：主要因为XXX功能上线/XXX活动拉动
- 新增下降：XXX渠道投放暂停

## 三、下周预测/关注点
- 预计XXX活动上线，日活有望提升
- 关注XXX渠道的投放效果
```

### 4. 存档方式
每周的周报按以下方式存档，方便日后检索：
```
/04-运营与内容/用户反馈周报/2025/2025-W33-用户反馈周报.docx
/05-市场与商务/渠道投放周报/2025/2025-W33-渠道投放周报.xlsx
/07-数据与策略/核心数据周报/2025/2025-W33-核心数据周报.docx
```
