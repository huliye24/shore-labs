# 资料存档问题



### 一、为什么要存档？（价值）

1.  **新人 onboarding**：新同事入职后，直接给ta一个文档链接，就能了解产品全貌、历史决策、代码规范。
2.  **问题追溯**：当出现线上故障或用户投诉时，能快速查到当时的代码版本、审核记录、决策过程。
3.  **合规与融资**：投资人做尽职调查时，需要看你的用户协议、隐私政策、内容审核记录、知识产权证明。这些资料必须随时拿得出来。

### 二、怎么存档？（极简体系）

初创公司不需要复杂的系统，**一个“公司云盘”+“在线文档工具”**就能解决大部分问题。

#### 1. 目录结构建议（以“岸”App为例）

在云盘或在线文档里建立如下文件夹：

```
/01-公司档案
  ├── 工商注册/投资文件（扫描件、合同）
  ├── 知识产权（商标注册证、软著证书）
  └── 员工合同/入职资料
/02-产品与设计
  ├── 00-版本发布记录（一个汇总文档，记录每个版本的发布时间、核心功能）
  ├── 01-需求文档（按版本号建子文件夹，如 V1.0、V1.1）
  ├── 02-原型图/设计稿（按版本号建子文件夹）
  └── 03-用户协议/隐私政策（存档历史版本，每次修改都保留一份）
/03-技术与代码
  ├── 架构图/数据库设计（只读，重要文档）
  ├── 接口文档（指向在线文档链接）
  └── 故障复盘报告（按时间命名，如 2025-08-15-服务器宕机复盘）
/04-运营与内容
  ├── 01-每日审核日志（按年月建子文件夹，如 2025/08/）
  ├── 02-举报处理台账（按年月建子文件夹）
  ├── 03-用户反馈汇总（按月存档）
  └── 04-活动方案/复盘（按活动名称建子文件夹）
/05-市场与商务
  ├── 合作合同（按合作方名称建子文件夹）
  └── 渠道投放数据（按月存档）
/06-内部管理
  ├── 周报汇总（按周建文件夹）
  ├── 会议纪要（按年月建文件夹）
  └── 关键决策备忘录（一个汇总文档，按时间记录）
```

#### 2. 命名规范（方便搜索）

-   **日期在前**：`20250815-服务器宕机复盘报告.docx`
-   **版本号清晰**：`岸-产品需求文档-V1.0.docx`，`岸-用户协议-20250801版.pdf`
-   **状态标注**：`需求池-进行中.xlsx`，`需求池-已完结-2025Q3.xlsx`（季度结束后把旧的需求池归档）

#### 3. 版本控制

-   **重要文件**：每次修改都“另存为”一份，不要直接覆盖。例如：`用户协议-20250101版`，`用户协议-20250601版`。
-   **代码**：用 Git 管理，本身就是最好的版本存档。

#### 4. 定期整理

-   **每月末**：运营同学把当月的审核日志、举报台账移到对应月份的文件夹。
-   **每季度末**：产品经理把本季度的需求文档归档，清空“进行中”的需求池，另存为“2025Q3已完结”。
-   **每次发版后**：把该版本的所有资料（需求文档、设计稿、测试报告）打包放进对应的版本文件夹。

### 三、几个关键点的存档建议

1.  **用户协议/隐私政策**：这是法律文件，每一次修改都必须存档旧版本，并记录修改日期和原因（比如：因为新出的法律法规，修改了第X条）。
2.  **内容审核日志**：**非常重要**。每天审核了多少条、屏蔽了多少、封禁了多少，这些数据最好导出为Excel或CSV文件，按月存档。万一遇到监管部门检查，这是你履行了审核义务的证明。
3.  **关键决策备忘录**：建议单独建一个文档，每次团队做重大决策（比如：放弃某个功能、调整收费模式、更换服务器供应商），简单记录一下：什么时间、谁参与的、讨论了什么、最终决定是什么、理由是什么。能避免以后扯皮。
4.  **员工离职交接清单**：员工离职时，让ta把工作相关的文档、代码位置、账号密码（如有）、未完成任务写在交接清单里，存档。这是公司资产，不是个人隐私。

### 四、工具推荐

-   **在线文档**：飞书文档、语雀。适合多人协作、实时更新，且自带历史版本。
-   **云盘**：阿里云盘、百度企业网盘、腾讯微云。适合存放设计稿原文件、合同扫描件、视频素材等大文件。
-   **代码**：GitHub、GitLab、Gitee。
-   **知识库**：如果团队发展到10人以上，可以考虑用飞书知识库，把散落的文档整合成结构化的知识库。



存档不是为了存档而存档，而是为了**让公司的经验可以积累，让后来的同事能站在前人的肩膀上**。

哪怕公司只有两三个人，养成随手存档、定期整理的习惯，会在未来省下无数时间。
