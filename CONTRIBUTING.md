# CONTRIBUTING

# 🤝 参与构建“岸” (Shore)

欢迎来到“岸”！这是一个拒绝内卷、拥护“存在感”的开源实验。我们正在寻找不仅仅是会写代码，更是对**人机关系、数字疗愈和极简设计**有共鸣的伙伴。

作为一名非技术背景的创始人，我负责产品的灵魂与架构，而你将是把这片“芦花荡”变为现实的魔法师。

------

### 🎨 我们欢迎哪些贡献？

我们不唯技术论，这里的协作是多维度的：

- **💻 核心开发：** 优化 FastAPI 后端、完善 Streamlit 交互或挑战 Three.js 的 3D 视觉表现。
- **✨ 艺术与 UI：** 持续打磨“玻璃拟态”视觉，加入更多梦幻的“奶油色”组件。
- **💡 产品哲学：** 参与《缺演论》与产品逻辑的碰撞，让功能更符合“上岸一刻，做回自己”的初心。
- **🛠️ 基础设施：** 帮助优化 AI 辅助开发的流程，让更多非技术志愿者也能参与进来。

------

### 🛠️ 如何开始？

1. **寻找灵感：** 查看仓库中的 `Issues`。我们标记了 `good first issue` 的任务非常适合初次尝试。
2. **开启对话：** 在 Issue 下留言你的想法。我们提倡“先交流，后动手”，确保大家在审美和逻辑上共频。
3. **提交代码 (PR)：** * 请保持代码的**模块化**（像积木一样易于拆卸和升级）。
   - 注释请尽量清晰，方便我们团队的其他成员（包括 AI 助手）理解。
4. **审美对齐：** 如果涉及 UI 改动，请参考 `Design_Guide.md`，我们追求的是极致的呼吸感，而非复杂的堆砌。

------

### 🌟 协作原则

- **存在重于竞争：** 这里没有 KPI。我们希望你的贡献是源于对产品的喜爱，而不是为了刷履历。
- **AI 友好：** 我们积极使用 Cursor 等 AI 工具。如果你也是 AI 辅助开发的深度用户，欢迎分享你的 Prompt 心得。
- **平等与透明：** 无论是 CEO 还是刚入校的大学生，在代码和灵感面前一律平等。

------

### 📜 行为准则

请保持友善与尊重。我们在这里是为了构建一个“缓冲地带”，所以请不要把压力和戾气带入社区。
