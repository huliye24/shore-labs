# 岸 (Shore) 2.0 开源项目

> **“岸”是一个关注「存在」而非「成长」的社交平台，旨在为用户提供一个不需要证明自己、无比较压力的空间。**

---

## 项目介绍

“岸”并不是一个传统的社交应用，它的核心理念是：

* **接住那些跑累的人。**
* **在脆弱发生时，是否能够被接住。**

在岸上，用户无需不断优化人设，也不需要高频创作内容，只需要确认自己的“存在”。我们拒绝“效率优先”模式，提供一个安静、无压力、充满理解的社交空间。

### 核心功能

1. **存在之墙**：用户发布一句话，简单地证明自己今天存在过，完全没有评论、点赞等社交互动。
2. **茶室**：可以发布评论，但评论内容默认被涂层覆盖，只有博主可以主动查看。这是一个保护用户隐私的设计。
3. **回声墙**：用户可以选择最多三条评论永久展示在自己的帖子下，作为纪念。所有回声墙评论都不可互动（不可回复、不可点赞、不可编辑、不可撤回）。
4. **我的回声**：用户的个人中心，收录被公开的回声评论，不显示数量，不进行排名。

这些功能的设计都是基于「存在」而非「优化」的原则。

## 开源与贡献

“岸”是一个开源项目，欢迎大家参与开发和改进。无论你是前端开发者、后端开发者、设计师，还是对这个项目理念感兴趣的贡献者，都可以帮助我们让这个产品走得更远。

### 项目目标

我们希望 **“岸”** 成为一个无比较压力、无成就追求的社交空间，帮助那些疲惫不堪的用户找到短暂的宁静和理解。如果你认同这个目标，欢迎你加入我们！

### 如何开始

1. **克隆仓库**：

   ```bash
   git clone https://github.com/your-organization-name/shoring-app.git
   cd shoring-app
   ```

2. **安装依赖**：

   根据项目的技术栈（例如 React、Node.js、Python 等），使用合适的包管理工具安装依赖。

   ```bash
   npm install   # 如果是 Node.js 项目
   # 或者
   pip install -r requirements.txt  # 如果是 Python 项目
   ```

3. **运行项目**：

   启动本地开发环境：

   ```bash
   npm start   # 启动前端
   # 或者
   python app.py   # 启动后端
   ```

## 设计理念与原则

“岸”的设计是基于以下原则：

* **无压力**：用户不需要证明自己，平台不推崇高效或成长。
* **可控的互动**：评论和互动必须受到保护，用户掌控自己的隐私和互动内容。
* **纪念性**：回声墙是一个“纪念区”，永不撤回，代表曾经的脆弱和真实。

我们希望 **“岸”** 能成为一个无压的空间，允许用户在不被迫“变好”的情况下，仅仅存在。
